SJTU jAccount 验证码助手

Features

为 jAccount 登录页量身打造

不胜频繁登录还要手输验证码的烦扰?交给它。

本地验证码识别

内置 ddddocr 的 ONNX 模型,通过 onnxruntime-web(WASM)在浏览器内推理,验证码图片不离开本机。

自动填写

监测登录页验证码出现或刷新,识别完成后自动填入输入框,无需任何手动操作。

可选自动登录

默认关闭。开启后账号密码经 AES-GCM 256 位加密保存在本机,打开登录页自动填充并提交。

克制的提交策略

仅当用户名、密码、验证码全部就绪时才延迟 500 ms 提交,每次页面加载最多尝试 3 次,不会无限循环。

最小权限

只申请本地存储权限和 jaccount.sjtu.edu.cn 的站点权限,不访问任何其它网站。

完全开源

源码以 MIT 许可证开源,构建与测试校验内置模型 SHA-256,代码与隐私政策公开可查。

How it works

三步告别手输验证码

安装扩展

从 Edge 商店一键安装,或用 GitHub Releases 的离线包加载。首次加载本地模型只需几秒。

打开登录页

访问 jAccount 登录页,扩展自动定位验证码图片,在本地识别并填入输入框。

(可选)自动登录

在弹窗中开启自动登录并加密保存账号密码,之后打开登录页即自动完成整个登录流程。

Privacy first

隐私优先:所有处理都在你的浏览器内完成

  • 零上传:不向开发者、第三方服务或任何外部服务器发送验证码、凭据或使用数据。
  • 加密存储:保存的账号密码经 AES-GCM 加密后仅存于本机,不可导出的密钥单独保存,可随时一键清除。
  • 无遥测:不使用分析、广告或遥测服务;扩展不发起任何外部网络请求。

详见完整的 隐私政策,或直接阅读 开源代码。

FAQ

常见问题

我的账号密码安全吗?

自动登录默认关闭。只有你主动开启并保存时,凭据才会经 Web Crypto 的 AES-GCM(256 位)加密后写入浏览器本地存储;加密密钥是不可导出的 CryptoKey,单独存于扩展的 IndexedDB。数据从不离开你的设备,开发者也无法访问。随时可在弹窗中点击「清除」,同时删除密文与密钥。

验证码识别错了怎么办?

页面验证码刷新后扩展会自动重新识别并填写。开启自动登录时,每次页面加载最多自动尝试 3 次,避免识别错误时无限循环;超过次数后你可以手动登录。

为什么扩展需要 WebAssembly(wasm-unsafe-eval)?

它是运行本地 ONNX 验证码识别模型所必需的,属于 Manifest V3 允许的 WASM 例外。所有模型与运行时均随扩展本地分发,不涉及任何远程代码。

会上架 Chrome Web Store 吗?

目前已上架 Microsoft Edge 外接程序商店。Chrome 用户可以使用 GitHub Releases 的离线包安装,后续如上架 Chrome Web Store 会在仓库中公告。
